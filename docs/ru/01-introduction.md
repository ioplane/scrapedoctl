# 01 - Введение

`scrapedoctl` — это профессиональная CLI-утилита и сервер Model Context Protocol (MCP), разработанный для обеспечения удобного взаимодействия AI-агентов с веб-контентом. Используя API [Scrape.do](https://scrape.do/), проект предоставляет мощный движок для скрапинга с обходом анти-бот систем и рендерингом JavaScript, оптимизированный как для человека, так и для машин.

## Цель проекта

Основная цель `scrapedoctl` — предоставить AI-агентам (таким как Claude Code, Gemini CLI и др.) высококачественные Markdown-представления веб-страниц, минимизируя при этом затраты за счёт персистентного кэширования и эффективного управления запросами.

## Основные возможности

- **Интерактивный REPL**: Оболочка в стиле Cisco с сокращениями команд, контекстной помощью (`?`) и автодополнением по Tab.
- **Мультипровайдерный веб-поиск**: Поиск в интернете через Scrape.do, ScraperAPI, SerpAPI (7 движков) или пользовательские exec-плагины.
- **Персистентное кэширование**: Встроенное хранилище SQLite для экономии токенов и ведения истории.
- **Интерфейс для машин**: Полная поддержка MCP (инструменты `scrape_url` и `web_search`) и JSON-метаданных для динамического обнаружения инструментов.
- **Обход анти-ботов**: Использование ротации прокси и браузерного рендеринга Scrape.do.
- **Версия и обновление**: Встроенная проверка релизов на GitHub через `scrapedoctl version` и поддержка самообновления.
- **Установка автодополнения**: Автоматическая XDG-совместимая установка для Bash, Zsh, Fish и PowerShell.
- **Современная архитектура**: Написано на Go 1.26, ядро без внешних зависимостей, строгий линтинг (golangci-lint v2).
- **Модуль PowerShell**: Модуль, соответствующий PSScriptAnalyzer, с нативным обнаружением бинарников для кроссплатформенного использования.
